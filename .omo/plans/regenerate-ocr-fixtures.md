# regenerate-ocr-fixtures — Regenerate OCR/AI verifier fixtures from real Gemini runs

## Background

Task 7 of `fix-agentic-engineering.md` was skipped because the Gemini API key hit daily quota during the validation run. The synthetic fixture files at `agentic_engineering/mocks/ocr_fixtures/` and `agentic_engineering/mocks/mock_ai_verifier.py` still contain placeholder data (e.g., "Indomie Goreng") rather than real OCR output from the dummy images.

## What needs to happen

1. Re-run fixture capture with a fresh/valid `GEMINI_API_KEY`:
   ```bash
   CAPTURE_FIXTURES=1 .venv/bin/python agentic_engineering/capture_fixtures.py
   ```
   This runs real OCR on `agentic_engineering/images/` and writes normalized JSON to `agentic_engineering/mocks/ocr_fixtures/`.

2. Run the AI verifier capture step in `capture_fixtures.py` to replace `mock_ai_verifier.py`'s hardcoded `["YES"]` with real Gemini responses for the ambiguous pairs generated by the matching pipeline.

3. Verify:
   ```bash
   MOCK_OCR=1 MOCK_AI_VERIFIER=1 bash prepare.sh && HAQITA_BATCH=1 ./haqita.sh && bash verify.sh --skip-sync
   ```
   OCR output should reflect real captured values, not synthetic ones.

4. Commit the updated fixture files.
